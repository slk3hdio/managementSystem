本项目旨在为牙科医院建立管理报表系统，其管理工作内容可从人员管理、资源管理、患者管理、流程管理、数据管理五个维度进行：
一、人员管理
明确医护及技术人员（医生、助理、洁牙师）的分工，如每张牙椅的人员配置及角色（诊疗、洁牙维保）；跟踪人员排班与工作状态（如 003 牙椅医生 “上午在总院” 的外出情况）。
二、资源管理（牙椅资源）
管理牙椅的使用分配，确保牙椅（诊疗类、洁牙维保类）的功能区分与高效利用；
统计牙椅使用率，通过 “是否就诊”“使用时长” 等数据评估资源利用率。
三、患者管理
分类管理患者（初诊、复诊、复查、洁牙维保），明确不同类别患者的收费规则；
跟踪患者预约、就诊流程，记录 “治疗项目”“预约人”“是否需客服跟进” 等信息，保障服务闭环。
四、流程管理
规范预约流程，明确前台、咨询、渠道、客服等不同预约发起方的权限；
把控诊疗流程，从 “预约 - 就诊 - 随访” 全环节跟踪，确保患者信息可追溯、服务有记录。
五、数据管理
日常填报管理日报表，记录患者、牙椅、人员的联动数据；
基于数据进行统计分析（如牙椅使用率、患者就诊率），为运营决策提供依据。

# 主要功能：

1. 人员管理模块
人员信息维护：支持医生、助理、洁牙师等角色的信息新增、编辑、删除，包含姓名、角色、联系方式等字段；
权限分配：根据角色（医生 / 助理 / 管理员）设置操作权限（如医生仅能查看自己的预约，管理员可全局管理）。
2. 牙椅管理模块
牙椅信息管理：维护牙椅编号、类型（诊疗 / 洁牙维保）等信息；
牙椅状态监控：实时标记牙椅 “可用 / 维护” 状态，支持状态变更记录。
3. 患者管理模块
患者档案管理：支持患者信息建档、查询、编辑，包含姓名、年龄、联系方式等基础信息；
患者分类管理：根据就诊类型（初诊 / 复诊 / 复查 / 洁牙）自动归类，关联历史就诊记录。
4. 预约管理模块（核心功能）
预约创建：支持选择患者、牙椅、接诊人员、预约时间、就诊类型、治疗项目、预约渠道等信息创建预约；
预约查询与调整：按日期、牙椅、人员等维度查询预约，支持预约时间、患者信息的修改与取消；
预约提醒：通过短信、系统消息等方式向患者和医护人员推送预约提醒；
就诊确认：医护人员标记患者 “是否就诊”“是否需客服跟进”，完善服务闭环。
5. 统计分析模块
牙椅使用率统计：自动计算每日 / 周 / 月牙椅实际使用时长与使用率，生成可视化报表；
患者就诊统计：分析不同就诊类型、渠道的患者占比，辅助运营决策；
人员工作统计：统计医护人员的接诊量、工作时长等数据，用于绩效考核参考。



# 数据字典：
1. 人员信息表（staff）
存储医生、助理、洁牙师等团队成员信息
字段名	类型	说明	约束
staff_id	INT	人员 ID（主键）	AUTO_INCREMENT
staff_name	VARCHAR(50)	姓名	NOT NULL

phone	VARCHAR(20)	联系电话	
create_time	DATETIME	创建时间	DEFAULT NOW()
2. 牙椅信息表（dental_chair）
存储牙椅配置及归属信息
字段名	类型	说明	约束
chair_id	INT	牙椅 ID（主键）	AUTO_INCREMENT
chair_num	VARCHAR(10)	牙椅编号（如 001/002）	UNIQUE NOT NULL
type	VARCHAR(30)	类型（诊疗 / 洁牙维保）	NOT NULL
status	TINYINT	状态（1 可用 / 0 维护）	DEFAULT 1
3. 患者信息表（patient）
存储患者基础信息
字段名	类型	说明	约束
patient_id	INT	患者 ID（主键）	AUTO_INCREMENT
patient_name	VARCHAR(50)	患者姓名	NOT NULL
phone	VARCHAR(20)	联系电话	
age	INT	年龄	
create_time	DATETIME	建档时间	DEFAULT NOW()
4. 预约记录表（appointment）
存储患者预约及就诊详情（核心表）
字段名	类型	说明	约束
appoint_id	INT	预约 ID（主键）	AUTO_INCREMENT
patient_id	INT	患者 ID（外键）	FOREIGN KEY
chair_id	INT	牙椅 ID（外键）	FOREIGN KEY
staff_id	INT	接诊人员 ID（外键）	FOREIGN KEY
appoint_date	DATE	预约日期	NOT NULL
start_time	TIME	开始时间	NOT NULL
end_time	TIME	结束时间	NOT NULL
visit_type	VARCHAR(30)	就诊类型（初诊 / 复查 / 洁牙）	NOT NULL
content	VARCHAR(200)	就诊内容（如 “多生牙复查”）	
booker	VARCHAR(50)	预约人	
book_channel	VARCHAR(30)	预约渠道（前台 / 咨询 / 渠道）	NOT NULL
is_visited	TINYINT	是否就诊（1 是 / 0 否）	DEFAULT 0
need_follow	TINYINT	是否需客服跟进（1 是 / 0 否）	DEFAULT 0
create_time	DATETIME	预约创建时间	DEFAULT NOW()
update_time DATETIME    更新时间
6. 牙椅使用率统计表（chair_usage）
存储每日牙椅使用统计数据
字段名	类型	说明	约束
usage_id	INT	统计 ID（主键）	AUTO_INCREMENT
chair_id	INT	牙椅 ID（外键）	FOREIGN KEY
stat_date	DATE	统计日期	NOT NULL
total_time	INT	总可用时长（分钟）	NOT NULL
used_time	INT	实际使用时长（分钟）	NOT NULL
usage_rate	DECIMAL(5,2)	使用率（used/total）	
create_time	DATETIME	统计时间	DEFAULT NOW()

# 前端设计

一、登录 / 权限页
1. 登录页
内容：账号输入框、密码输入框、角色选择（管理员 / 医生 / 前台）；
设计：简洁的品牌展示（医院 logo）、背景图贴合牙科主题，登录框居中布局，支持记住密码功能。
2. 权限首页（工作台）
按角色展示个性化工作台：
管理员：系统概览（今日预约数、牙椅使用率、待跟进患者数）、快捷入口（人员管理、牙椅配置、报表统计）；
医生：今日预约日程、患者档案快捷查询、已接诊患者统计；
前台：预约创建 / 修改入口、今日预约列表、患者建档入口。
二、基础信息管理页
1. 人员管理页
布局：左侧筛选栏（角色类型、入职时间），右侧列表展示（人员 ID、姓名、角色、电话、操作按钮）；
功能区：新增人员（弹窗表单：姓名、角色、电话等）、编辑 / 删除 / 详情查看按钮；
设计：支持表格排序、搜索，列表行区分角色颜色（如医生 - 蓝色、助理 - 绿色）。
2. 牙椅管理页
布局：牙椅卡片式展示（按编号排列），每张卡片包含牙椅编号、类型、状态；
功能区：新增牙椅（弹窗表单）、状态切换（可用 / 维护）、编辑详情按钮；
3. 患者管理页
布局：顶部搜索栏（姓名 / 电话），下方患者列表（ID、姓名、电话、年龄、建档时间、操作按钮）；
功能区：新建患者（弹窗表单）、查看病历 / 就诊记录、编辑 / 删除按钮；
设计：支持按建档时间筛选，列表可导出 Excel。
三、核心业务操作页
1. 预约管理页（核心）
布局：
顶部：日期选择器、“新建预约” 按钮；
主区域： 显示预约表格， 横轴为牙椅，纵轴为时间段，单元格内显示患者，牙椅，牙医，治疗项目等；
功能区：新建预约（弹窗表单：患者选择、牙椅选择、时间选择、就诊类型等）、编辑预约、标记 “是否就诊”“需跟进”，支持通过点击单元格进行预约或修改；
2. 预约详情页
内容：患者信息（姓名、电话、年龄）、预约信息（牙椅、时间、就诊类型、治疗内容）、接诊人员、预约渠道、就诊状态、跟进记录；
四、数据统计分析页
1. 牙椅使用率统计页
布局：
顶部：时间筛选（日 / 周 / 月）、牙椅筛选；
主区域：使用率趋势图表（折线图 / 柱状图）、数据表格（牙椅编号、总时长、使用时长、使用率）；
设计：图表支持交互（点击查看详情），数据支持导出。
2. 患者就诊统计页
内容：就诊类型占比饼图、预约渠道统计柱状图、患者增长趋势图；
功能：支持按时间、牙椅、医生筛选数据，图表可下载。
3. 人员工作统计页
内容：医生接诊量排行、助理配合次数统计、工作时长对比图；
设计：按角色切换统计维度，数据表格与图表联动。
五、系统设置页
1. 权限设置页
内容：角色列表、权限分配表单（勾选菜单权限 / 操作权限）；
功能：新增角色、编辑权限、重置密码。
2. 基础配置页
内容：就诊类型管理（新增 / 编辑 / 删除类型）、预约渠道管理、系统参数设置（如牙椅每日可用时长）；
设计：表单式布局，配置项分组展示。
六、辅助页
1. 消息通知页
内容：预约提醒、待跟进患者通知、系统公告；
设计：按时间排序，未读消息标红，支持标记已读 / 删除。
2. 帮助中心页
内容：操作指南（图文 / 视频）、常见问题 FAQ、联系客服入口；
设计：分类导航，搜索功能。
页面组织逻辑
层级导航：顶部主导航（工作台、基础管理、预约管理、统计分析、系统设置）+ 侧边栏子导航（如基础管理下分人员 / 牙椅 / 患者）；
操作闭环：从列表页可快速进入详情页 / 编辑页，操作后返回原页面并刷新数据；
响应式设计：适配电脑端 / 平板端，列表页在小屏设备下自动折叠次要列，保留核心信息。
